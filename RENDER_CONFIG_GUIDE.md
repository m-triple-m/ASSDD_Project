# Quick Start Configuration for Render.com

## Files Generated for Deployment:

1. **requirements.txt** - Python package dependencies
   - Contains Flask, PyTorch, OpenCV, and other dependencies
   - Automatically used by Render during build

2. **render.yaml** - Render.com configuration
   - Specifies build and start commands
   - Configures environment variables
   - Links to PostgreSQL database

3. **Procfile** - Alternative process file
   - Contains the command to start the application
   - Compatible with multiple platforms

4. **runtime.txt** - Python version specification
   - Specifies Python 3.11.5
   - Ensures consistent Python version across deployments

5. **.env.example** - Environment variable template
   - Copy to `.env` for local development
   - Do NOT commit `.env` file to git

6. **build.sh** - Build preparation script
   - Creates necessary directories
   - Checks for model file
   - Can be extended for model download

## Next Steps:

### Step 1: Create GitHub Repository
```bash
git add .
git commit -m "Add Render.com deployment configuration"
git push origin main
```

### Step 2: Set Up Render Service
1. Visit https://dashboard.render.com
2. Connect your GitHub repository
3. Create a new Web Service
4. Use these settings:
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn app:app`

### Step 3: Configure Environment Variables
In Render Dashboard → Environment:
- Set `SECRET_KEY` to a secure random string (or use Render's auto-generate)
- Database URL will be auto-set if you create PostgreSQL service

### Step 4: Handle Model File
The model file is large (150+ MB) and in .gitignore.
Choose ONE approach:

**Option A: Upload via Render Dashboard** (Smallest downtime)
1. Download build artifacts after first deployment
2. Use Render's Files feature to upload model

**Option B: Use External Storage** (Recommended for large files)
1. Upload model to AWS S3, Google Cloud Storage, or similar
2. Modify build.sh to download during deployment

**Option C: Git LFS** (If < 1GB)
1. Install Git LFS: `git lfs install`
2. Track file: `git lfs track "*.pth"`
3. Commit and push

## Performance Considerations:

- PyTorch inference may be slow on CPU-only plans
- Consider GPU plan if available for your account tier
- Optimize image processing pipeline for faster response times
- Implement caching for frequently processed images
